============================================================
0. 文稿定位
============================================================

本文不替代第二编 R1.0，而是在 R1.0 已经给出的正式主线对象基础上，补足进入第三编前必须澄清的接口问题。

R1.0 已经完成的关键工作是：

    C → Ready(C) → ApplySet(C) → e
    e → ≺
    (𝔈, ≺) → Σ_l
    𝒰(e), ℐ(e) → 第三编 S_upd / S_int 来源

其中事件对象已经压实为：

    e = (U_e, R_e, W_e, X_e)

U_e 表示事件内实际执行的更新单元集合；
R_e 表示读集；
W_e 表示写集；
X_e 表示拒绝、冲突或共同 Reject 痕迹集合。

本文的任务不是增加新的物理假设，而是修正以下接口风险：

1. 直接依赖关系与传递因果偏序混用；
2. 因果链长度 ℓ_c 若直接在传递闭包上取最短链，可能退化；
3. “最大反链”与“极大反链”术语混用；
4. Ready(C) 中“当前层”“已接受配置”可能偷渡外部时间；
5. (𝔈, ≺) 尚未显式要求局部有限性；
6. N5 条件仍偏自然语言，尚未完全谓词化。

============================================================
1. R1.1 保留的 R1.0 主线
============================================================

R1.1 不改变 R1.0 的基本链条：

    C → Ready(C) → ApplySet(C) → e = (U_e, R_e, W_e, X_e)

也不改变第二编交付给第三编的基本对象方向：

    事件集合 𝔈
    严格因果偏序 ≺
    因果链长度 ℓ_c
    因果切片 Σ_l
    更新痕迹集合 𝒰(e)
    干涉关系 ⋈_C
    干涉痕迹集合 ℐ(e)

R1.1 只做接口修补：把这些对象之间的关系写得更严格，使第三编不能重新发明“时间”“空间”或“邻接”。

============================================================
2. 新增：直接结构依赖关系 →_D
============================================================

R1.0 中直接使用 e_i ≺ e_j 表示事件间结构依赖。R1.1 将其拆成两层：

    e_i →_D e_j

表示 e_i 是 e_j 的直接结构依赖；

    e_i ≺ e_j

表示 e_i 在严格因果偏序中先于 e_j。

定义 2.1（直接结构依赖）。

    e_i →_D e_j

当且仅当至少满足以下条件之一：

1. e_i 的写集进入 e_j 的读集：

    W_{e_i} ∩ R_{e_j} ≠ ∅

2. e_i 与 e_j 存在写冲突，并且该冲突的消解决定进入 e_j 的形成条件：

    W_{e_i} ∩ W_{e_j} ≠ ∅

3. e_i 的执行结果进入 e_j 的 Ready 判定：

    U_{e_i} ∩ DepReady(e_j) ≠ ∅

4. e_i 与 e_j 共享拒绝、冲突或共同 Reject 痕迹，并且该痕迹影响 e_j 的合法性：

    X_{e_i} ∩ X_{e_j} ≠ ∅

5. e_i 出现在 e_j 的对照链、消解链或局域合法性证明中。

定义 2.2（严格因果偏序）。

严格因果偏序 ≺ 定义为直接结构依赖关系 →_D 的传递闭包：

    e_i ≺ e_j  ⇔  e_i →_D^+ e_j

因此：

    →_D 是直接依赖边；
    ≺ 是严格因果可达关系。

============================================================
3. 严格因果偏序的条件
============================================================

R1.1 要求：

1. 反自反性：

    ¬(e ≺ e)

2. 传递性：

    e_i ≺ e_j 且 e_j ≺ e_k  ⇒  e_i ≺ e_k

3. 无环性：

    不存在 e_1 ≺ e_2 ≺ ... ≺ e_1

如果直接依赖图 (𝔈, →_D) 出现有向环：

    e_1 →_D e_2 →_D ... →_D e_1

则该环不能直接进入正式因果偏序。必须采取二者之一：

方案 A：合并为一个宏事件；

    E_macro = {e_1, ..., e_n}

方案 B：判定为未消解冲突，拒绝形成并列事件。

裁决原则：

    因果环不能进入正式严格因果偏序。

============================================================
4. 新增：局部有限性
============================================================

R1.0 已要求 U_e、R_e、W_e、X_e 与 ApplySet(C) 为有限集合。R1.1 进一步要求因果结构本身局部有限：

    ∀ e_i ≺ e_j,
    |{q ∈ 𝔈 : e_i ≺ q ≺ e_j}| < ∞

含义是：

    任意两个因果相关事件之间，只能有有限多个中间事件。

由此，第二编交付给第三编的对象不再只是普通偏序，而是：

    (𝔈, ≺) = 局部有限严格偏序事件集

这一步是从 BICFT 更新结构进入因果集型离散时空接口的必要条件。

============================================================
5. 修正：因果链长度 ℓ_c
============================================================

R1.0 将 ℓ_c 表述为“偏序图上的最短因果链长度”。若 ≺ 已经是传递闭包，则直接在 ≺ 上取最短链可能退化为 1。因此 R1.1 先定义覆盖关系。

定义 5.1（覆盖关系 / Hasse 边）。

    e_i ⋖ e_j

当且仅当：

    e_i ≺ e_j

且不存在 q 使得：

    e_i ≺ q ≺ e_j

也就是说，e_i ⋖ e_j 是 Hasse 图中的直接因果边。

定义 5.2（最短 Hasse 链长）。

    ℓ_min(e_i,e_j)
    =
    min { n : e_i=e_0 ⋖ e_1 ⋖ ... ⋖ e_n=e_j }

它适用于第三编的 r_c-因果邻域构造。

定义 5.3（最长因果链长）。

    ℓ_max(e_i,e_j)
    =
    max { n : e_i=e_0 ≺ e_1 ≺ ... ≺ e_n=e_j }

它更适合表示因果深度。

定义 5.4（高度函数）。

    h(e)
    =
    max { n : e_0 ⋖ e_1 ⋖ ... ⋖ e_n=e }

若 e 没有前驱，则：

    h(e)=0

R1.1 建议：

    第三编 r_c-邻域使用 ℓ_min；
    因果深度、层级分析与切片候选使用 h 或 ℓ_max；
    不再把所有用途都压到一个 ℓ_c 上。

============================================================
6. 修正：最大反链改为极大反链或切片族
============================================================

R1.0 中“最大反链”的定义是：在不破坏反链性质的前提下不能再加入新的事件。

该定义更准确地称为：

    极大反链（maximal antichain）

而不是：

    最大反链（maximum antichain）

R1.1 建议将 Σ_l 改写为：

    Σ_l = 极大反链切片

定义 6.1（反链）。

    Σ ⊂ 𝔈

若对任意不同事件 e_i,e_j ∈ Σ，有：

    ¬(e_i ≺ e_j)
    ¬(e_j ≺ e_i)

则 Σ 是反链。

定义 6.2（极大反链）。

Σ 是极大反链，当且仅当 Σ 是反链，且不存在 q ∉ Σ 使得：

    Σ ∪ {q}

仍为反链。

定义 6.3（高度层）。

    A_l = { e ∈ 𝔈 : h(e)=l }

由高度函数定义可推出：

    A_l 是反链。

证明要点：

若 e_i,e_j ∈ A_l 且 e_i ≺ e_j，则 h(e_j) ≥ h(e_i)+1，与 h(e_i)=h(e_j)=l 矛盾。

定义 6.4（切片族）。

若切片不唯一，则不强行指定唯一 Σ_l，而定义：

    𝔖_l = { Σ : Σ 是极大反链，且 A_l ⊆ Σ }

第三编可以从 𝔖_l 中选取承载层，但选取协议只能依赖：

    ≺, ⋖, h, 𝒰(e), ℐ(e)

不得依赖外部标签、文件顺序、候选编号或算法扫描顺序。

============================================================
7. 修正：Ready(C) 改为 Ready(C | D)
============================================================

R1.0 中 Ready(C) 的条件包括“已接受配置”和“当前层”。R1.1 将其改写为相对于因果下闭集 D 的判定：

    Ready(C | D) ∈ {0,1}

其中 D ⊂ 𝔈 是因果下闭集：

    e ∈ D 且 q ≺ e  ⇒  q ∈ D

含义是：

    在已经形成的因果过去 D 下，候选配置 C 是否可形成事件。

Ready(C | D)=1 当且仅当：

1. C 的读集对象在 D 所确定的可访问结构中存在；
2. C 的前置结构依赖已经包含于 D；
3. C 与 D 中已形成事件不存在不可消解写冲突；
4. C 不违反第一编无背景红线；
5. C 满足由 D 与局域规则共同给出的局域合法性条件。

裁决原则：

    D 是因果过去，不是外部时间片。
    事件形成顺序只是构造用线性扩展，不得成为物理结构。

标签无关性要求：

若不同构造顺序 Order_1 与 Order_2 都满足同一组局域规则，则最终得到的因果结构必须同构：

    (𝔈_1, ≺_1) ≅ (𝔈_2, ≺_2)

若不能同构，则说明算法顺序进入了物理结构，需要回退修正。

============================================================
8. N5 条件谓词化
============================================================

R1.0 中 N5 表示至少两个同时 ready 的候选配置之间存在非平凡冲突关系，或存在不可约局域改写分歧。R1.1 将其写成谓词。

定义 8.1（N5）。

    N5(C_i,C_j | D)=1

当且仅当：

    Ready(C_i | D)=1
    Ready(C_j | D)=1

并且至少满足以下条件之一：

1. 写-写冲突：

    WriteSet(C_i) ∩ WriteSet(C_j) ≠ ∅

2. 写-读冲突：

    WriteSet(C_i) ∩ ReadSet(C_j) ≠ ∅

3. 读-写冲突：

    WriteSet(C_j) ∩ ReadSet(C_i) ≠ ∅

4. ApplySet 互斥：

    ApplySet(C_i) ∩ ApplySet(C_j) ≠ ∅

且二者不能被视为同一事件的合并执行；

5. RejectTrace 共享：

    RejectTrace(C_i) ∩ RejectTrace(C_j) ≠ ∅

6. 二者在对照链中不可独立判定。

定义 8.2（冲突裁决）。

    Resolve(C_i,C_j | D)
    ∈ { {C_i}, {C_j}, {C_i,C_j}_merge, ∅ }

分别表示：

    保留 C_i；
    保留 C_j；
    合并为宏事件；
    二者均拒绝。

Resolve 必须只依赖局域结构，不依赖候选编号、文本顺序或扫描顺序。

10. 第三编 Past/Future 与 r_c-邻域的接口定义
============================================================

给定事件 e ∈ 𝔈：

    Past(e) = { q ∈ 𝔈 : q ≺ e }

    Future(e) = { q ∈ 𝔈 : e ≺ q }

半径受限版本：

    Past_r(e) = { q : q ≺ e, ℓ_min(q,e) ≤ r }

    Future_r(e) = { q : e ≺ q, ℓ_min(e,q) ≤ r }

因果邻域：

    N_r^≺(e) = Past_r(e) ∪ Future_r(e) ∪ {e}

注意：

    r_c 是因果链半径；
    r_s 若出现，应是切片内距离代理半径；
    r_c 不得与空间距离半径混用。

============================================================
11. 切片内空间邻接的接口约束
============================================================

第三编若要在 Σ_l 或某个 Σ ∈ 𝔖_l 上构造空间邻接，不能直接假设“相邻”。必须从第二编对象导出。

一个允许的邻接代理可以写成：

    S_r(e_i,e_j)
    =
    α |Past_r(e_i) ∩ Past_r(e_j)| / |Past_r(e_i) ∪ Past_r(e_j)|
    +
    β |Future_r(e_i) ∩ Future_r(e_j)| / |Future_r(e_i) ∪ Future_r(e_j)|
    +
    γ |ℐ(e_i) ∩ ℐ(e_j)| / |ℐ(e_i) ∪ ℐ(e_j)|

其中：

    e_i,e_j ∈ Σ
    ¬(e_i ≺ e_j)
    ¬(e_j ≺ e_i)

定义切片内邻接：

    e_i ~_{Σ,r,θ} e_j  ⇔  S_r(e_i,e_j) ≥ θ

于是第三编的空间承载图可写为：

    G_Σ = (Σ, ~_{Σ,r,θ})

这一步的意义是：

    空间邻接不是外加的，
    而是由共同过去、共同未来与干涉痕迹诱导出来的。

12. 第二编 R1.1 的三个定理雏形
============================================================

定理 1：事件生成导出局部有限严格偏序。

若：

1. 每个事件 e 由有限的 U_e,R_e,W_e,X_e 构成；
2. 直接依赖关系 →_D 只由读写依赖、Ready 依赖、ApplySet 结果、RejectTrace 与对照链依赖导出；
3. (𝔈, →_D) 无有向环；
4. 任意因果区间中只有有限多个中间事件；

则：

    (𝔈, ≺),  ≺ = →_D^+

是局部有限严格偏序事件集。

定理 2：高度层是反链。

定义：

    h(e)=max { n : e_0 ⋖ e_1 ⋖ ... ⋖ e_n=e }

则：

    A_l={e:h(e)=l}

是反链。

定理 3：极大反链切片可以作为离散空间承载候选。

若 Σ 是包含高度层 A_l 的极大反链，则 Σ 中任意两个不同事件互不构成因果先后。因此 Σ 可以作为第三编空间承载候选。

但：

    Σ 不是空间几何本身。

空间几何还需第三编从 Past/Future、𝒰(e)、ℐ(e) 与切片内邻接代理继续构造。

14. R1.1 裁决
============================================================

R1.1 的目标不是推进到物质场，也不是直接证明连续极限，而是先把第二编交付给第三编的接口对象修正为：

    𝔠_2 =
    (𝔈, ≺, ⋖, ℓ_min, ℓ_max, h, 𝔖_l, 𝒰, ⋈_C, ℐ)

由此，第三编可以合法构造：

    Past/Future
    r_c-因果邻域
    极大反链承载层
    切片内邻接代理
    离散几何承载图

当前主线应保持为：

    更新配置
    → 事件
    → 直接结构依赖
    → 严格因果偏序
    → 局部有限因果事件集
    → 极大反链切片族
    → 第三编离散几何接口
