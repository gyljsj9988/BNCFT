**N=8 / Xi_k(alpha,beta) / D_eff / 工程阈值 / 区域分类**

版本定位：v0.4 是数值封版准备包，不是最终数值封版。它把 A2 冻结源包拆成可回填、可审计、可验收的最小单元。未取得 A2 原始数值、完整 Xi_k 公式和正式区域标签前，不允许补写、猜测或经验拟合。

## A3.0 v0.4 的封版目标

A3 v0.4 的任务是把 A2 已冻结内容整理成六个可回填源包，并给出每个源包的字段、入口、出口、证据链和验收门控。v0.4 不产生新阈值，不新增区域，不改写 Xi_k(alpha,beta)，不替换 D_eff 默认算法。

v0.4 完成后，A3 v0.5 可以直接按包回填：先填 N=8 索引，再填 Xi_8 逐槽定义，再填 D_eff 默认算法，再填 A2 工程阈值，再填区域树，最后填证据项和审计结果。

## A3.5 A2-DEFF 默认算法包回填卡

A2-DEFF 包必须把 D_eff 默认算法写成可执行契约：输入是什么，前置条件是什么，步骤如何排列，何时输出，何时未决，何时回退。A3 v0.4 不允许引入替代 D_eff 算法。

| 步骤 | 名称 | 输入 | 通过 | 未决 | 回退 |
| --- | --- | --- | --- | --- | --- |
| DEFF-0 | 输入检查 | Xi_8、A2-N8、A2-TH | Xi_8 完整且有限 | 输入不全但可补齐 | Xi_8 不可定义或不可修复 |
| DEFF-1 | 默认聚合/计算 | Xi_8 | 按 A2 默认算法执行 | 算法参数待核验 | 算法前提失败 |
| DEFF-2 | 稳定性检查 | D_eff 候选值 | 满足 A2 稳定阈值 | 接近边界带 | 非有限、奇异或失稳 |
| DEFF-3 | 输出约束 | D_eff | 单位/范围/精度均满足 A2 | 精度或单位待确认 | 输出与 A2 约束冲突 |
| DEFF-4 | 证据绑定 | 计算记录 | 可复算且可追溯 | 缺少复核记录 | 无法复现 |

```text
D_eff_DEFAULT_CONTRACT(Xi_8):
    require N == 8
    require Xi_8 is complete under A2-XI
    if any forced fallback in Xi_8: return fallback
    if any pending band in Xi_8: mark pending
    compute D_eff by A2-DEFF default algorithm
    apply A2-TH stability and validity gates
    if any fallback gate: return fallback
    if any pending gate: return pending with D_eff?
    return pass with D_eff
```

## A3.9 状态聚合规则

v0.4 固化三态聚合纪律：回退优先于未决，未决优先于通过。只有当所有门控均通过，且区域分类唯一时，最终状态才允许写作通过。

| 条件 | 状态 | 输出纪律 |
| --- | --- | --- |
| 存在任一强制回退项 | 最终状态 = 回退 | 不得继续给出正式 D_eff 或正式区域标签 |
| 不存在回退，但存在任一未决项 | 最终状态 = 未决 | 可列候选值，但不得写成结论 |
| 所有门控通过且区域唯一 | 最终状态 = 通过 | 输出 D_eff、区域标签与证据项 |
