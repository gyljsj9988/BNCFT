1. 第三编的基本红线
============================================================

第三编不得直接引入：

    外部时间 t；
    预设空间坐标 x；
    预设空间距离 d(x,y)；
    连续流形 M；
    先验维数 d；
    先验度规 g_{μν}。

第三编只能使用第二编交付的对象：

    𝔈, ≺, ⋖, ℓ_min, ℓ_max, h, 𝔖_l, 𝒰, ⋈_C, ℐ

并从这些对象中导出：

    Past/Future；
    因果邻域；
    切片；
    切片内邻接；
    空间承载图；
    距离代理；
    体积代理；
    维数代理；
    稳定性判据。

裁决原则：

    空间不是输入；
    空间承载层必须从因果偏序与事件痕迹中导出。

============================================================
2. Past / Future / Interval
============================================================

定义 2.1（因果过去）。

对任意事件 e ∈ 𝔈：

    Past(e) = { q ∈ 𝔈 : q ≺ e }

定义 2.2（因果未来）。

    Future(e) = { q ∈ 𝔈 : e ≺ q }

定义 2.3（因果区间 / Alexandrov 型区间）。

若 e_i ≺ e_j，则定义：

    I(e_i,e_j)
    =
    { q ∈ 𝔈 : e_i ≺ q ≺ e_j }

局部有限性要求：

    |I(e_i,e_j)| < ∞

第三编中任何基于区间计数、邻域计数或体积代理的构造，都必须先通过局部有限性检查。

============================================================
3. r_c-因果邻域
============================================================

第三编需要构造局域邻域，但不能直接假设空间球。因此先定义因果半径。

定义 3.1（过去 r_c-邻域）。

    Past_r(e)
    =
    { q ∈ 𝔈 : q ≺ e 且 ℓ_min(q,e) ≤ r }

定义 3.2（未来 r_c-邻域）。

    Future_r(e)
    =
    { q ∈ 𝔈 : e ≺ q 且 ℓ_min(e,q) ≤ r }

定义 3.3（双向因果邻域）。

    N_r^≺(e)
    =
    Past_r(e) ∪ Future_r(e) ∪ {e}

注意：

    r 是因果链半径；
    r 不是空间距离半径；
    r 不得直接解释为连续时空中的空间球半径。

若需要壳层结构，定义：

    Shell_r^-(e)
    =
    { q ∈ 𝔈 : q ≺ e 且 ℓ_min(q,e)=r }

    Shell_r^+(e)
    =
    { q ∈ 𝔈 : e ≺ q 且 ℓ_min(e,q)=r }

壳层比完整邻域更适合构造局域相似度，因为完整邻域可能受到深层共同过去的过度影响。

============================================================
4. 因果高度层与极大反链切片族
============================================================

第二编 R1.1 中定义高度函数：

    h(e)
    =
    max { n : e_0 ⋖ e_1 ⋖ ... ⋖ e_n=e }

若 e 没有前驱，则 h(e)=0。

定义 4.1（高度层）。

    A_l = { e ∈ 𝔈 : h(e)=l }

命题 4.1：

    A_l 是反链。

证明：

若 e_i,e_j ∈ A_l 且 e_i ≺ e_j，则存在一条从 e_i 到 e_j 的因果链，因此 h(e_j) ≥ h(e_i)+1，与 h(e_i)=h(e_j)=l 矛盾。

因此同一高度层中的事件互不构成因果先后。

定义 4.2（极大反链切片）。

Σ 是极大反链，当且仅当：

1. Σ ⊂ 𝔈；
2. 任意不同 e_i,e_j ∈ Σ 满足：

       ¬(e_i ≺ e_j)
       ¬(e_j ≺ e_i)

3. 不存在 q ∉ Σ 使得 Σ ∪ {q} 仍为反链。

定义 4.3（切片族）。

    𝔖_l
    =
    { Σ : Σ 是极大反链，且 A_l ⊆ Σ }

第三编不应假设唯一空间切片。若切片不唯一，则可计算对象应尽量对 𝔖_l 内的切片选择稳定。

裁决原则：

    Σ 是空间承载候选；
    Σ 不是空间几何本身。
    空间几何必须进一步由切片内邻接与距离代理导出。

============================================================
5. 切片内空间邻接的基本问题
============================================================

在同一极大反链切片 Σ 中，任意两个事件 e_i,e_j 满足：

    ¬(e_i ≺ e_j)
    ¬(e_j ≺ e_i)

这只表示二者因果不可比。

但“因果不可比”不等于“空间相邻”。

因此第三编必须解决的问题是：

    如何在不预设空间坐标的情况下，
    从共同过去、共同未来、更新痕迹和干涉痕迹中
    推出切片内空间邻接？

这就是第三编的核心接口问题。

============================================================
6. 共同因果邻域相似度
============================================================

定义一个基础 Jaccard 函数：

    J(A,B)
    =
    |A ∩ B| / |A ∪ B|,   若 |A ∪ B| > 0

    J(A,B)=0,            若 |A ∪ B|=0

不把两个空集的相似度设为 1，是为了避免边界事件或信息不足事件被误判为高度相似。

定义 6.1（过去壳层相似度）。

给定权重 w_k ≥ 0 且 Σ_{k=1}^r w_k = 1：

    C_r^-(e_i,e_j)
    =
    Σ_{k=1}^r w_k ·
    J(Shell_k^-(e_i), Shell_k^-(e_j))

定义 6.2（未来壳层相似度）。

    C_r^+(e_i,e_j)
    =
    Σ_{k=1}^r w_k ·
    J(Shell_k^+(e_i), Shell_k^+(e_j))

定义 6.3（干涉痕迹相似度）。

    C^I(e_i,e_j)
    =
    J(ℐ(e_i), ℐ(e_j))

定义 6.4（更新痕迹相似度，可选）。

若第三编需要使用更新痕迹，可以定义：

    C^U(e_i,e_j)
    =
    J(𝒰(e_i), 𝒰(e_j))

但 𝒰(e) 的使用要小心：更新痕迹相似不必然表示空间邻近，它也可能表示同类过程在远处重复出现。

因此 R1.0 建议主邻接代理优先使用：

    共同过去 + 共同未来 + 干涉痕迹

而把 𝒰(e) 作为辅助项。

============================================================
7. 切片内总相似度
============================================================

对 e_i,e_j ∈ Σ，定义：

    S_{Σ,r}(e_i,e_j)
    =
    α C_r^-(e_i,e_j)
    +
    β C_r^+(e_i,e_j)
    +
    γ C^I(e_i,e_j)
    +
    δ C^U(e_i,e_j)

其中：

    α,β,γ,δ ≥ 0

并且：

    α+β+γ+δ = 1

默认建议：

    δ = 0 或很小；
    α 与 β 作为主项；
    γ 用于引入干涉结构；
    δ 仅用于调试或增强验证线。

若只使用因果结构，不使用痕迹，则可取：

    γ=δ=0
    α+β=1

裁决原则：

    S_{Σ,r} 是空间邻接代理，不是物理距离本身。

============================================================
8. 切片内邻接协议
============================================================

为了避免阈值选择导致图过密或过稀，第三编 R1.0 建议使用“互为局域近邻 + 最低相似度”的邻接协议。

定义 8.1（k-近邻集合）。

对 e ∈ Σ，令：

    K_k(e)

表示 Σ \ {e} 中按照 S_{Σ,r}(e,q) 排序最高的 k 个事件集合。

若存在并列，可由局域结构稳定规则裁决；不得使用事件编号、文本顺序或扫描顺序裁决。

定义 8.2（切片内邻接）。

    e_i ~_{Σ,r,k,θ} e_j

当且仅当：

1. e_i,e_j ∈ Σ；
2. e_i ≠ e_j；
3. e_i ∈ K_k(e_j)；
4. e_j ∈ K_k(e_i)；
5. S_{Σ,r}(e_i,e_j) ≥ θ。

于是第三编得到空间承载图：

    G_Σ
    =
    (Σ, ~_{Σ,r,k,θ})

其中：

    Σ 是节点集合；
    ~_{Σ,r,k,θ} 是从因果邻域与痕迹相似度导出的边关系。

若不希望引入 k，也可使用纯阈值协议：

    e_i ~ e_j  ⇔  S_{Σ,r}(e_i,e_j) ≥ θ

但纯阈值协议容易在高度均匀或高度纠缠结构中生成过密图。因此正式主线建议优先使用互为 k-近邻协议。

============================================================
9. 空间距离代理
============================================================

定义 9.1（无权图距离）。

在 G_Σ 上，定义：

    d_Σ^0(e_i,e_j)
    =
    从 e_i 到 e_j 的最短边数

若不可达，则：

    d_Σ^0(e_i,e_j)=∞

定义 9.2（加权边长）。

若需要加权距离，可定义每条边的长度：

    L(e_i,e_j)
    =
    1 / (ε + S_{Σ,r}(e_i,e_j))

其中 ε>0 是防止除零的正小量。

定义 9.3（加权图距离）。

    d_Σ^w(e_i,e_j)
    =
    所有从 e_i 到 e_j 的路径中边长和的最小值。

裁决原则：

    d_Σ^0 与 d_Σ^w 都只是空间距离代理；
    它们不是连续度规；
    连续度规只能在后续粗粒化与极限检验中讨论。

============================================================
10. 体积代理
============================================================

因果集型结构中，计数是最原始的体积代理。

定义 10.1（事件计数体积）。

对 A ⊂ 𝔈：

    Vol_E(A) = |A|

对 B ⊂ Σ：

    Vol_Σ(B) = |B|

若后续引入基本体积尺度 v_0，则可写：

    Vol_phys(A) = v_0 |A|

但第三编 R1.0 不强行引入 v_0。

定义 10.2（更新加权体积，可选）。

若事件规模差异必须保留，可定义：

    Vol_U(A)
    =
    Σ_{e∈A} ω(e)

其中：

    ω(e)=1

是默认选择；或：

    ω(e)=f(|𝒰(e)|, |R_e|, |W_e|, |X_e|)

但更新加权体积会引入额外模型选择，正式主线默认先使用事件计数体积。

============================================================
11. 空间球、增长率与维数代理
============================================================

在 G_Σ 上定义空间球：

    B_Σ(e,R)
    =
    { q ∈ Σ : d_Σ^0(e,q) ≤ R }

定义 11.1（局部空间维数代理）。

给定 R_1<R_2：

    D_s(e;R_1,R_2)
    =
    [ log |B_Σ(e,R_2)| - log |B_Σ(e,R_1)| ]
    /
    [ log R_2 - log R_1 ]

若在某个尺度窗口内：

    D_s(e;R_1,R_2)

对 e 与 R 的变化较小，则称该切片在此窗口存在空间维数平台。

定义 11.2（切片平均空间维数）。

    D_s(Σ;R_1,R_2)
    =
    Avg_{e∈Σ} D_s(e;R_1,R_2)

定义 11.3（因果邻域增长维数代理）。

    D_c(e;r_1,r_2)
    =
    [ log |N_{r_2}^≺(e)| - log |N_{r_1}^≺(e)| ]
    /
    [ log r_2 - log r_1 ]

其中 N_r^≺ 是因果邻域，不是空间球。

第三编应同时保留：

    D_s：切片内空间图维数代理；
    D_c：因果邻域增长维数代理。

二者一致或稳定，是后续连续极限候选的重要信号；二者不一致，则提示该结构可能不是简单流形型几何。

============================================================
12. 粗糙度与局域稳定性代理
============================================================

定义 12.1（局域球增长比）。

    ρ_Σ(e;R)
    =
    |B_Σ(e,R+1)| / |B_Σ(e,R)|

若 |B_Σ(e,R)|=0，则该量不定义。

定义 12.2（切片粗糙度）。

    Rough(Σ;R)
    =
    Var_{e∈Σ} ρ_Σ(e;R)

粗糙度越大，表示不同事件周围的局域图增长差异越大。

定义 12.3（度数粗糙度）。

    DegRough(Σ)
    =
    Var_{e∈Σ} deg_{G_Σ}(e)

若使用互为 k-近邻协议，度数仍可能变化，因为互选约束会移除部分边。因此度数粗糙度仍是有意义的稳定性指标。

============================================================
13. 切片选择稳定性
============================================================

由于极大反链切片不唯一，第三编必须检查几何对象是否依赖任意切片选择。

给定两个同层或邻近层切片：

    Σ, Σ' ∈ 𝔖_l

或：

    Σ ∈ 𝔖_l,  Σ' ∈ 𝔖_{l+1}

需要比较以下不变量或准不变量：

1. 度数分布；
2. 球增长函数 |B_Σ(e,R)|；
3. 维数代理 D_s；
4. 粗糙度 Rough；
5. 连通分量数量；
6. 局域邻接模式。

定义 13.1（切片稳定性分数，形式占位）。

    Stab(Σ,Σ')
    =
    1
    -
    Dist(
        Inv(G_Σ),
        Inv(G_{Σ'})
      )

其中 Inv(G_Σ) 表示图不变量集合，Dist 表示归一化差异。

正式实现中 Dist 可以使用：

    分布距离；
    谱距离；
    图拉普拉斯谱差异；
    局域球增长差异；
    粗粒化后结构差异。

裁决原则：

    若几何量强依赖切片选择，
    则该几何对象不能作为无背景主线对象；
    只能作为某个 gauge-like 选片下的辅助描述。

============================================================
14. 粗粒化接口
============================================================

第三编还应为第四编粗粒化留出接口。

给定空间承载图：

    G_Σ=(Σ,~)

定义一个块划分：

    Π_Σ = {B_1,B_2,...,B_m}

其中：

    B_a ⊂ Σ
    B_a ∩ B_b = ∅
    ∪_a B_a = Σ

定义 14.1（块邻接）。

    B_a ~ B_b

当且仅当存在 e_i ∈ B_a, e_j ∈ B_b，使得：

    e_i ~ e_j

定义 14.2（块体积）。

    Vol(B_a)=|B_a|

或使用更新加权体积：

    Vol_U(B_a)=Σ_{e∈B_a}ω(e)

定义 14.3（块间因果关系）。

对不同切片中的块 B_a, B_b，定义：

    B_a ≺_B B_b

当且仅当存在 e_i∈B_a, e_j∈B_b，使得：

    e_i ≺ e_j

并且该关系在块级别无环，或已经通过宏块合并消除环。

粗粒化必须保持：

    局部有限性；
    无因果环；
    切片稳定性；
    不依赖外部标签。

============================================================
15. 向场论接口开放的最小结构
============================================================

第三编不直接完成物质场理论，但它需要交出后续可定义场的承载结构。

给定事件集合 𝔈，可定义事件场：

    φ : 𝔈 → ℝ

给定切片 Σ，可定义切片场：

    φ_Σ : Σ → ℝ

在空间承载图 G_Σ 上，可定义图拉普拉斯：

    (Δ_Σ φ)(e)
    =
    Σ_{q ~ e} w_{eq} [φ(q)-φ(e)]

其中 w_{eq} 可取：

    w_{eq}=1

或：

    w_{eq}=S_{Σ,r}(e,q)

也可以使用加权边长的倒数。

这只是第四编或物质场接口的准备，不等于完整粒子理论。

裁决原则：

    第三编只交出可承载场的离散几何结构；
    粒子、传播子、规范场与费米子不在第三编 R1.0 中强行完成。

============================================================
16. 离散几何有效性的通过条件
============================================================

第三编 R1.0 的最小通过条件如下。

P1. 因果输入有效：

    (𝔈,≺) 是局部有限严格偏序；
    ⋖、ℓ_min、h 可计算。

P2. 切片有效：

    至少存在一个极大反链切片 Σ；
    若切片不唯一，必须保留切片族 𝔖_l 或给出无标签选取协议。

P3. 邻接有效：

    切片内邻接 ~_{Σ,r,k,θ} 只能由因果邻域与痕迹相似度导出；
    不得使用外部空间坐标。

P4. 图结构有效：

    G_Σ 不能在主尺度上完全空图；
    也不能在主尺度上退化为近似完全图；
    度数分布应存在可控范围。

P5. 维数有效：

    在某些尺度窗口内，D_s 或 D_c 应存在稳定平台。

P6. 切片稳定：

    关键几何量对 𝔖_l 内合理切片选择不应剧烈变化。

P7. 标签无关：

    所有裁决不得依赖事件编号、文本顺序、候选扫描顺序或实现顺序。

若 P1-P7 失败，则第三编不能声称已经得到离散几何承载层，只能声称得到某个依赖选取协议的辅助图结构。

17. 最小玩具例：1+1 维因果菱形格
============================================================

为了验证第三编接口是否真正从因果结构导出空间邻接，给出一个最小玩具例。

定义事件集合：

    𝔈 = { (t,x) : 0≤t≤T, -L≤x≤L, t+x 为偶数 }

定义直接因果边：

    (t,x) ⋖ (t+1,x-1)
    (t,x) ⋖ (t+1,x+1)

只保留落在有限窗口内的事件。

严格因果偏序 ≺ 是 ⋖ 的传递闭包。

高度函数为：

    h(t,x)=t

因此高度层为：

    A_l = { (l,x) : l+x 为偶数 }

A_l 是反链，可作为切片 Σ_l。

考虑同一切片中的两个事件：

    e=(l,x)
    e'=(l,x+2)

它们没有因果先后关系。

但它们共享一个一阶过去事件：

    (l-1,x+1)

并共享一个一阶未来事件：

    (l+1,x+1)

因此：

    C_1^-(e,e') 较高；
    C_1^+(e,e') 较高；
    S_{Σ,1}(e,e') 较高。

对于更远的事件：

    e''=(l,x+4)

它与 e 不共享一阶过去或一阶未来，因此：

    S_{Σ,1}(e,e'') 较低。

使用 r=1 的互为 k-近邻协议后，Σ_l 上的邻接图恢复为一维链状结构：

    (l,x-2) ~ (l,x) ~ (l,x+2)

这说明：

    空间邻接可以由共同过去与共同未来诱导，
    不需要预设空间坐标。

这个玩具例是第三编 R1.0 的最小单元测试。

19. 町机制与内生局域性
============================================================

第三编需要进一步说明“为什么某些切片上的事件会自然聚成局域邻近块”。

“町”可以理解为由相似度高、相互作用强的事件簇，
它们在切片内表现为高密度、低粗糙度的邻接子图。

定义 19.1（町集）。

    𝒯 = { T_a ⊂ Σ : T_a 连通于 G_Σ }

其中连通性可以基于 G_Σ 的边关系或基于更严格的 S_{Σ,r} 阈值连通性。

町之间的边界称为町壁：

    町壁是连接不同 町子图的边集，
    这些边通常对应较低的 S_{Σ,r} 值。

町机制的内生来源应满足：

- 同一町内事件的共同过去/未来相似度高于跨町事件；
- 干涉痕迹 ℐ(e) 与更新痕迹 𝒰(e) 在町内更容易重叠；
- 町壁对应的边在 k-近邻互选后易于被剪除。

如果能够从事件本身的痕迹与因果邻域定义出町结构，则 K 近邻不再是纯人工设定，而是“内生邻近性的自然极限”。

20. 参数扫描与验证协议
============================================================

为了让第三编的结果具有可重复性，必须给出明确的参数扫描与验证协议。

参数集合包括：

- r：因果壳层半径；
- k：互为 k-近邻的邻接宽度；
- θ：最低相似度阈值；
- α, β, γ, δ：相似度权重；
- 切片选择策略：从 𝔖_l 中选取 Σ 或多个 Σ。

建议的参数扫描流程：

1. 取若干相邻高度层 l，并从 𝔖_l 中选取代表性切片；
2. 在 r∈{1,2,3}、k∈{2,3,4,5}、θ∈[0,0.9] 的网格上扫描；
3. 对每组参数计算：
   - 平均度数；
   - 连通分量数；
   - D_s(Σ;R_1,R_2)；
   - Rough(Σ;R)；
   - Stab(Σ,Σ') 与切片间变异；
   - 町分布与町壁强度；
4. 判定稳定平台：
   - 若参数变化较小导致上述量保持在同一数量级，则认为该参数区间具有稳定性；
   - 否则视为非几何区间。

推荐的权重选择：

- α≈β≈0.4；
- γ≈0.1；
- δ≈0 或 0.05；

当 α 与 β 同时占优时，结果主要依据共同过去与共同未来；
当 γ 增大时，可观察干涉痕迹对“町结构”的增强效果。

21. 失败模式与反例库
============================================================

为了说明第三编的界限，必须明确失败模式并建立反例库。

失败模式包括：

- 纯随机偏序：
  S_{Σ,r} 在同一切片内呈近似均匀分布，G_Σ 退化为近似完全图；
- 明星型节点：
  某些事件与大量事件都有高相似度，使得 G_Σ 出现高阶度中心，破坏局域性；
- 町壁消失：
  事件聚类无法形成清晰边界，导致所有事件都处于同一弱连接大簇；
- 低对称无几何性：
  切片间稳定性差异极大，切片选择结果成为几何的主导因素。

反例库应至少包含：

- 纯随机偏序样本；
- 结构化但非局域的事件生成过程；
- 含重复更新痕迹但无因果邻域对应的样本；
- 仅以 𝒰(e) 为相似度的虚假几何样本。

每个反例都应对应一个失败判据，例如：

- D_s 不存在平台；
- Rough 与 DegRough 极大；
- Stab(Σ,Σ') 低于可接受阈值；
- 町壁强度接近随机噪声水平。

这些反例有助于区分“真正的离散几何承载层”与“形式化的图结构”。

25. 伪代码与实现框架

为了避免第三编停留在概念层面，必须给出明确的算法框架，使得后续实现可直接对接。

25.1 伪代码：切片内相似度和邻接图构造

输入：

- 𝔈, ≺, ⋖, ℓ_min, h, 𝔖_l, ℐ, 𝒰（若可用）
- 参数：r, k, θ, α, β, γ, δ
- 目标切片 Σ ∈ 𝔖_l

输出：

- G_Σ = (Σ, E_Σ)
- S_{Σ,r}(e_i,e_j)

伪代码：

    function ComputeShell(e, r, direction):
        if direction == 'past':
            return { q ∈ 𝔈 : q ≺ e and ℓ_min(q,e) == r }
        else if direction == 'future':
            return { q ∈ 𝔈 : e ≺ q and ℓ_min(e,q) == r }

    function Jaccard(A, B):
        if |A ∪ B| == 0: return 0
        return |A ∩ B| / |A ∪ B|

    function Similarity(e_i, e_j, Σ, r, α, β, γ, δ):
        C_minus = Σ_{m=1}^r w_m * Jaccard(ComputeShell(e_i,m,'past'), ComputeShell(e_j,m,'past'))
        C_plus = Σ_{m=1}^r w_m * Jaccard(ComputeShell(e_i,m,'future'), ComputeShell(e_j,m,'future'))
        C_I = Jaccard(ℐ(e_i), ℐ(e_j))
        C_U = Jaccard(𝒰(e_i), 𝒰(e_j))
        return α*C_minus + β*C_plus + γ*C_I + δ*C_U

    function BuildSliceGraph(Σ, r, k, θ):
        for each e in Σ:
            neighbors[e] = []
        for each unordered pair (e_i,e_j) in Σ:
            s = Similarity(e_i,e_j, Σ, r, α, β, γ, δ)
            S_{Σ,r}(e_i,e_j) = s
            record score for e_i and e_j
        for each e in Σ:
            K_k(e) = top-k events q ∈ Σ\{e} by S_{Σ,r}(e,q)
        E_Σ = {}
        for each unordered pair (e_i,e_j) in Σ:
            if e_j ∈ K_k(e_i) and e_i ∈ K_k(e_j) and S_{Σ,r}(e_i,e_j) >= θ:
                E_Σ.add({e_i,e_j})
        return Graph(Σ, E_Σ)

25.2 伪代码：图结构与维数代理

输入：G_Σ, R_1, R_2

输出：D_s(Σ;R_1,R_2), Rough(Σ;R), DegRough(Σ)

伪代码：

    function GraphBall(G, e, R):
        return { q ∈ Σ : shortest_path_length(G, e, q) <= R }

    function ComputeDimension(G, R_1, R_2):
        for each e in Σ:
            B1 = GraphBall(G, e, R_1)
            B2 = GraphBall(G, e, R_2)
            if |B1| > 0 and |B2| > 0:
                D_s(e) = (log(|B2|) - log(|B1|)) / (log(R_2) - log(R_1))
            else:
                D_s(e) = undefined
        return average of defined D_s(e)

    function ComputeRoughness(G, R):
        ratios = []
        for each e in Σ:
            B_R = GraphBall(G, e, R)
            B_R1 = GraphBall(G, e, R+1)
            if |B_R| > 0:
                ratios.append(|B_R1| / |B_R|)
        return variance(ratios)

    function ComputeDegreeRoughness(G):
        degrees = [deg(e) for e in Σ]
        return variance(degrees)

25.3 伪代码：切片稳定性与参数扫描

输入：切片族 𝔖_l, 参数集合 P
输出：Stab 数据、稳定参数区间

    function CompareInvariants(G1, G2):
        return normalized_distance(
            InvariantVector(G1),
            InvariantVector(G2)
        )

    function ScanParameters(Σ, params):
        results = []
        for r in params.r_values:
            for k in params.k_values:
                for θ in params.θ_values:
                    G = BuildSliceGraph(Σ, r, k, θ)
                    D = ComputeDimension(G, params.R_1, params.R_2)
                    rough = ComputeRoughness(G, params.R)
                    degrough = ComputeDegreeRoughness(G)
                    results.append((r,k,θ,D,rough,degrough,G))
        return results

    function AssessSliceStability(Σ_set, params):
        stability = []
        for Σ in Σ_set:
            results = ScanParameters(Σ, params)
            stability.append(results)
        for each pair Σ_i, Σ_j in Σ_set:
            score = CompareInvariants(G_i, G_j)
            record stability score
        return stability

22. K 近邻物理化与町机制
============================================================

当前报告揭示的关键路径：

- 机制1：K 近邻关系能够把谱维数从 1209 拉回小整数；
- 机制2：纯因果生长尚未自动涌现低维；
- 机制3：双阱势下局域关系能够保留拓扑结。

这说明 BNCFT 的独特切入点在于“町”这一物理机制。

定义 22.1（町壁局域势差）。

在事件层面，引入一个“局域势差函数” F(e_i,e_j)，表征两事件是否属于同一个町：

- 若 e_i、e_j 共享大量局域更新痕迹和干涉痕迹，则 F(e_i,e_j) 大；
- 若它们穿过町壁，则 F(e_i,e_j) 小。

建议把 S_{Σ,r}(e_i,e_j) 扩展为：

    S_{Σ,r}(e_i,e_j) = α C_r^-(e_i,e_j) + β C_r^+(e_i,e_j) + γ C^I(e_i,e_j)
                      + η · H(e_i,e_j)

其中 H(e_i,e_j) 是“町亲和度”，由局域双阱/更新势信息自发给出。

若 H 可以从事件的“共同局域过程”中定义，则 K 近邻规则就不再是手设，而是由系统内在结构决定。

### 22.1 町机制的候选来源

H(e_i,e_j) 的候选来源包括：

- 共同满足同一局域稳定条件的事件对；
- 共享同一“双阱基元”或“町核心子集”的事件；
- 被同一类局域张量/场配置影响的事件；
- 间接由相同干涉痕迹集成的事件。

### 22.2 物理化的目标

目标不是把 H 设为一个新坐标，而是把它定义为：

    H(e_i,e_j) = F(ℐ(e_i),ℐ(e_j),𝒰(e_i),𝒰(e_j),Past(e_i)∩Past(e_j),Future(e_i)∩Future(e_j))

即只用现有接口对象。

这样一来，空间邻接就可以表示为“同一町内的因果不可比事件”，而不是“同一坐标层的事件”。

24. 失败模式与稳定性检验
============================================================

第三编应明确以下三类失败模式，并用它们指导参数选择与机制设计：

24.1 空图失败

若 S_{Σ,r}(e_i,e_j) 对大多数 e_i,e_j 都太小，则互为 k-近邻协议可能产生一个几乎空的 G_Σ。
这种情况下：
- 空间图不连通；
- 球增长过慢；
- 度数分布集中在 0 或很小整数。

诊断：
- 共同过去/未来重叠太小；
- 干涉痕迹或町亲和度未提供足够的局域联系；
- θ 设定过高或 k 设定过小。

24.2 全连图失败

若 S_{Σ,r}(e_i,e_j) 对多数事件对都很大，则 G_Σ 可能接近完全图。
这种情况下：
- 空间图丧失局域性；
- 谱维数倾向于极大值；
- 体积增长与距离代理失去物理意义。

诊断：
- 相似度定义过于宽泛；
- 事件间共同过去/未来或痕迹重叠过多；
- KNN 稀疏化无效，互选约束被突破；
- 未引入町/势差机制。

24.3 星状图失败

当部分节点与大多数节点连接，而节点间连接稀少时，G_Σ 会退化为星状结构。
这种情况下：
- 度数分布高度不均；
- 球增长结果极不稳定；
- 维数代理受中心节点主导。

诊断：
- S_{Σ,r} 过度依赖一类强相似度；
- KNN 互选造成“明星节点”现象；
- 部分事件具有异常高的共同过去/未来。

24.4 稳定性检验

对于每种失败模式，第三编应给出可计算的检验量：

- `ConnFrac(G_Σ)`：最大连通分量占比；
- `AvgDeg(G_Σ)`：平均度数；
- `DegVar(G_Σ)`：度数方差；
- `D_s(Σ;R_1,R_2)` 的标准差；
- `Stab(Σ,Σ')`：不同切片间的几何不变量差异。

若这些检验量在参数范围内保持稳定，则该结构具备继续进入粗粒化与场论接口的资格。
否则，应先回到“相似度定义 + 町机制”阶段调整。

25. 继续推进的决策原则
============================================================

当前阶段的合理推进顺序：

1. 优先完成“离散几何承载层”的内部稳定性；
2. 再考虑“町机制”如何具体物理化 K 近邻；
3. 只有当 G_Σ 的几何判据稳定后，才进入第四编的粗粒化与场论接口；
4. 不要在第三编阶段硬性追求连续度规或粒子谱。

理由很简单：

    若空间承载图不稳定，
    后续的场、粒子和连续极限都会悬空。

这意味着第三编的真正目标不是“直接构造时空”，而是“构造一个稳定的、可验证的离散几何接口”。

26. 最小玩具例与反例库
============================================================

第三编应同时建立最小玩具例和反例库，以便用具体实例校准接口。

26.1 玩具例：1+1 维因果菱形格

- 事件集合：

      𝔈 = { (t,x) : t=0,1,...,T; x=-t,-t+2,...,t }

- 直接因果覆盖关系：

      (t,x) ⋖ (t+1,x±1)

- 严格因果偏序 ≺ 是 ⋖ 的传递闭包。
- 高度函数： h(t,x)=t。
- 同一高度层 A_t 作为切片 Σ_t。

取 r=1、k=2、θ 适中，S_{Σ,r} 由一阶过去与未来相似度给出，得到的 G_Σ 恢复为链状邻接结构。

该玩具例验证：
- 共同过去/未来本身就能产生空间邻接；
- 该模型不需要外部坐标来区分相邻与远离；
- 维数代理 D_s(Σ;1,2) ≈ 1。

26.2 反例库

反例库应包括至少三类结构：

1. 无局域性的随机偏序；
2. 由纯阈值相似度产生的全连或近全连图；
3. 由高阶共同过去产生的“明星节点”星状图。

反例库的用途：
- 验证 S_{Σ,r} 与 ~_{Σ,r,k,θ} 的稳健性；
- 校准 failure mode 的诊断规则；
- 防止后续实现因实验效果误判为“稳定几何”。

26.3 最低实现测试集

建议至少准备以下测试集：
- 1+1 维因果菱形格；
- 1+1 维有限随机偏序（局部有限但非流形形态）；
- 1+1 维“全连接+随机切片”结构；
- 2+1 维潜变量预设的局域因果网。

这些测试集可以逐步增强为：
- 2+1 维纯因果生长模型；
- 近似町结构的双阱势示例；
- 非平坦切片下的稳定性测试。

27. 第三编通过条件清单
============================================================

第三编通过条件应明确写成清单，便于后续审核。

27.1 因果输入通过条件

- (𝔈, ≺) 是局部有限严格偏序；
- h(e)、ℓ_min(e_i,e_j) 和 𝔖_l 可计算；
- 极大反链切片族 𝔖_l 不依赖外部标签选择。

27.2 空间承载通过条件

- 至少存在一个切片 Σ 使得 G_Σ 连通且度数在有限范围内；
- G_Σ 在参数变动下不出现空图/全连图/星状图；
- D_s(Σ;R_1,R_2) 在合理窗口内稳定；
- Stab(Σ,Σ') 不超过预设阈值。

27.3 町与拓扑通过条件

- 若输入包括町相关痕迹，则 H(e_i,e_j) 可定义为现有对象的函数；
- 町壁对应的邻接弱化；
- 同一町内事件比跨町事件更易邻接。

27.4 实现与验证通过条件

- 提供至少一个玩具例成功恢复预期几何；
- 提供反例库并演示失败模式；
- 给出稳定性检验量并展示其在选定参数范围内的收敛/稳定行为。

以上通过条件构成第三编 R1.0 的最小验收框架。

28. 未来验证路线
============================================================

28.1 数值验证路线

- 用 Toy 输入对 S_{Σ,r}、H、~_{Σ,r,k,θ} 进行扫描；
- 记录：平均度、连通度、球增长、D_s、Stab；

28.2 理论验证路线

- 证明：若事件产生于具有局域性潜变量的隐空间，则 S_{Σ,r} 可恢复邻接；

28.3 进入第四编的条件

- 粗粒化后仍应保持因果无环性和切片选取稳定性；
- 第四编接口宜从 G_Σ 和 Π_Σ 出发，不再依赖原始事件标签。
